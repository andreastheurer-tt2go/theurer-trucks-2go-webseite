# Slack-Bot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Einen Slack-Bot aufsetzen, der Rückfragen zu den Ads-Wochenberichten im Thread oder per DM beantwortet, mit Zugriff auf historische Reports aus einer Postgres-Datenbank auf dem Hostinger-VPS.

**Architecture:** Postgres-Container als neuer Docker-Service auf dem bestehenden VPS. Bestehender Wochenbericht-Workflow (n8n) schreibt Raw-Data + KI-Bericht in Postgres. Neuer n8n-Workflow empfängt Slack-Events, verifiziert Signaturen, lädt letzte 8 Reports pro Plattform, fragt Claude API und antwortet in Slack (Thread oder DM).

**Tech Stack:** Docker (bestehend), PostgreSQL 16, n8n (bestehend), Slack Events API, Anthropic API (Sonnet 4.6)

**Spec:** `docs/specs/2026-04-19-slack-bot-design.md`

-- Tabelle für Wochenberichte aller Werbeplattformen (Google Ads, Meta Ads, ...)

CREATE TABLE IF NOT EXISTS ads_reports (
  id                SERIAL PRIMARY KEY,
  platform          VARCHAR(50)  NOT NULL,           -- 'google_ads' | 'meta_ads'
  week_start        DATE         NOT NULL,
  week_end          DATE         NOT NULL,
  raw_data          JSONB        NOT NULL,           -- Komplette Script-Rohdaten

  created_at        TIMESTAMPTZ  DEFAULT NOW(),
  CONSTRAINT uq_platform_week UNIQUE (platform, week_start)
);

CREATE INDEX IF NOT EXISTS idx_platform_week
  ON ads_reports (platform, week_start DESC);

-- Kommentare zur Dokumentation
COMMENT ON TABLE ads_reports IS 'Wöchentliche Ads-Performance-Berichte aller Plattformen';
COMMENT ON COLUMN ads_reports.platform IS 'Plattform-Bezeichner: google_ads, meta_ads';
COMMENT ON COLUMN ads_reports.raw_data IS 'Komplette Rohdaten vom jeweiligen Script';
COMMENT ON COLUMN ads_reports.ai_report IS 'Claude-generierter Wochenbericht (Slack-Markdown)';

- [ ] **Step 4: Code-Node "Signature verifizieren"**

Node hinzufügen → "Code" (JavaScript):

```javascript
// Slack Signature Verification (HMAC-SHA256)
// https://api.slack.com/authentication/verifying-requests-from-slack

const crypto = require('crypto');

const signingSecret = $env.SLACK_SIGNING_SECRET;
if (!signingSecret) {
  throw new Error('SLACK_SIGNING_SECRET nicht gesetzt');
}

const input = $input.first();
const headers = input.json.headers || {};
const timestamp = headers['x-slack-request-timestamp'];
const slackSignature = headers['x-slack-signature'];

// Raw Body aus Webhook-Node (Raw Body Option muss aktiv sein)
// Der Body kommt als Buffer in $binary.data
const rawBodyBuffer = input.binary?.data?.data
  ? Buffer.from(input.binary.data.data, 'base64')
  : null;

if (!rawBodyBuffer || !timestamp || !slackSignature) {
  return [{ json: { valid: false, reason: 'Fehlende Headers oder Body' } }];
}

const rawBody = rawBodyBuffer.toString('utf8');

// Replay-Attack-Schutz: Requests älter als 5 Minuten ablehnen
const now = Math.floor(Date.now() / 1000);
if (Math.abs(now - parseInt(timestamp, 10)) > 300) {
  return [{ json: { valid: false, reason: 'Timestamp zu alt' } }];
}

// Signatur berechnen und vergleichen (timing-safe)
const baseString = `v0:${timestamp}:${rawBody}`;
const expectedSig = 'v0=' + crypto
  .createHmac('sha256', signingSecret)
  .update(baseString)
  .digest('hex');

const sigA = Buffer.from(expectedSig);
const sigB = Buffer.from(slackSignature);
const valid = sigA.length === sigB.length && crypto.timingSafeEqual(sigA, sigB);

// Geparsten Body für nachfolgende Nodes bereitstellen
const parsedBody = JSON.parse(rawBody);

return [{
  json: {
    valid,
    body: parsedBody,
    rawBody,
  },
}];
```

Node-Name: `Signatur prüfen`

- [ ] **Step 7: Code-Node "Event-Typ bestimmen"**

An TRUE-Ausgang von "Gültig?" anhängen → "Code" (JavaScript):

```javascript
const body = $input.first().json.body;

// Slack URL-Verification (einmalig beim Event-Subscriptions-Setup)
if (body.type === 'url_verification') {
  return [{
    json: {
      respondType: 'challenge',
      challenge: body.challenge,
    },
  }];
}

// Event-Callback: app_mention oder message.im
if (body.type === 'event_callback' && body.event) {
  const event = body.event;

  // Bot-eigene Nachrichten ignorieren (Endlosschleife vermeiden)
  if (event.bot_id || event.subtype === 'bot_message') {
    return [{ json: { respondType: 'ignore' } }];
  }

  // Nur app_mention oder DM zulassen
  const isAppMention = event.type === 'app_mention';
  const isIm = event.type === 'message' && event.channel_type === 'im';

  if (!isAppMention && !isIm) {
    return [{ json: { respondType: 'ignore' } }];
  }

  // Frage-Text bereinigen (Bot-Mention entfernen)
  const cleanText = (event.text || '').replace(/<@[A-Z0-9]+>\s*/g, '').trim();

  return [{
    json: {
      respondType: 'process',
      event: {
        user: event.user,
        channel: event.channel,
        channel_type: event.channel_type,
        ts: event.ts,
        thread_ts: event.thread_ts || null,
        text: cleanText,
        isIm,
      },
    },
  }];
}

return [{ json: { respondType: 'ignore' } }];
```

Node-Name: `Event-Typ bestimmen`

  .filter(m => m.ts !== currentTs)

  .map(p => {

${platformList}

- Formatiere für Slack (Slack-Markdown: *fett*, _kursiv_, • Bullet Points)
- Max 2000 Zeichen pro Antwort
- Sei konkret und actionable, nicht generisch
- Wenn die Daten für die Frage nicht reichen: klar sagen
- Emojis sparsam aber sinnvoll (:chart_with_upwards_trend:, :warning:, :white_check_mark:, :mag:, :moneybag:)`;

  userMessage += `## Plattform: ${platform}\n\n`;

    userMessage += `### Woche ${r.week_start} bis ${r.week_end}\n`;
    userMessage += `**KI-Bericht:**\n${r.ai_report}\n\n`;

  }
}

}

  json: {
    systemPrompt,
    messages,
    channel: event.channel,
    thread_ts: event.thread_ts || event.ts,

  },
}];
